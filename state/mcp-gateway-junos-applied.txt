## Topology (IPsec bypassed; Tailscale used instead)

  LLM (llm-gateway gpt-oss-120b) -> mcp-gateway (zrok share "mcp-gateway")
    -> http backend over Tailscale -> hermes jmcp (100.105.37.124:30030)
    -> NETCONF/SSH -> SRX345-IM (192.168.0.254)

  Why not the srx-tunnel IPsec path: the SRX is behind a modem whose IPsec ALG
  breaks NAT-T (IKE remapped to :1025, no UDP-4500 float, IKE_AUTH reply never
  returns). Modem cannot be changed. Tailscale (WireGuard) sidesteps it.

## Caveat (operational)

  OCI <-> hermes Tailscale is a DERP relay (double-NAT, no direct link) and flaps.
  mcp-gateway holds one backend connection per client session and does NOT
  auto-reconnect; after a flap, restart the dialer:
    sudo systemctl restart mcp-access-junos-gateway
  Prefer execute_junos_command (single RPC) over gather_device_facts.
